# Einrichtung

Um in diesem Kurs die praktischen Aufgaben zu lösen, ist es nötwendig die Arbeitsumgebung einzurichten. Hierzu benötigen Sie Programme auf dem System mit welchen Sie arbeiten werden. Da die Einrichtung während des ersten Praktikums durchgeführt wird, dient dies hier lediglich als Kurzanleitung.

Visual Studio Code (VSC, VS-Code)

VS-Code ist ein Texteditor mit welchem die enstsprechenden Codezeilen geschrieben werden können. VSC ist im Bereich der Webentwicklung der Industriestandard. Durch seine Schlankheit ist VSC nicht Performancelastig und gleichzeitig kann es durch Erweiterungen so angepasst werden, dass es für alle Bereiche der Entwicklung passend ist.

Installation

Sie finden VSC hier. Mit einem Klick auf den Pfeil (rot unterstrichen in Abbildung) können Sie die MacOs (Apple-Mac) Version herunterladen. Achten Sie darauf, immer den Stable-Build zu nehmen.
Anschließend führen Sie die Installation auf Ihrem System durch. Es wird geraten das Programm auf englisch zu belassen

Kurzeinweißung

Ihr VSC wird nach der Installation nicht genauso aussehen, da auf dem Bild entsprechende Extensions schon installiert wurden und das Programm und über die Jahre angepasst wurde.

Empfohlene Extensions

Bei einem Klick auf das Extensions-Icon auf der linken Seite , öffnet sich ein Menü mit dem Erweiterungen gesucht und installiert werden können, folgende Erweiterungen werden empfohlen, sind aber nicht notwendig für das abschließen der praktischen Arbeiten:

Github

Github ist eine Plattform für die Versionsverwaltung. Für die genauere Beschreibung von Github wenden Sie sich bitte an den Professor oder einem Betreuer. Die Verwendung von solchen Portalen wird im digitalen Bereich oft vorausgesetzt, weshalb es auch zum Standard gehört. Die Vorteile der Benutzung werden hier kurz erläutert, hiebei wird für die bessere Vertändlichkeit nicht zwischen Git und Github unterschieden.

# Name Beschreibung
1 Sicherheit Durch Github haben Sie jederzeit den aktuellen-, und alle vergangenen Versionen Ihres Projektes. Falls Sie durch einen technischen Ausfall nicht mehr auf die Inhalte zugreifen können, können Sie über die Plattform immer den aktuellen Stand klonen. Außerdem ist es möglich ältere Stände zu laden, falls bei der Entwicklung etwas schief ging. Zusätzlich erlaubt es Ihnen von mehr als einem PC am Projekt zu arbeiten und beim welchsel immer auf den aktuellen Stand zu springen
2 Teamarbeit An Projekten wird überlicherweiße nicht alleine gearbeitet sondern im Team. Durch Git lassen sich ein Projekt in Branches teilen, mit welchen eine Gruppe an einem Bereich arbeiten kann während ein anderes Team an einem anderen Bereich arbeitet. Dies wird für das Praktikum nicht relevant

Bewertungssystem

Die Abgaben im praktischen Teil sind die Vorausgesetzung für die Durchführung des praktischen Endabgabe. Da die Endabgabe deutlich mehr von den Studierenden abverlangt als die Aufgaben selbst, wird am einem Bewertungssystem gemessen, in wie weit Sie sich für diese Abgabe qualifiziert haben. Beim Erfüllen der jeweiligen Arbeiten, werden Ziele festgelegt welche Sie durch den praktischen Teil erarbeiten sollen. Hierbei ist die Erfüllung dieser Ziele das Bewertungskriterium an dem Sie gemessen werden.
Für jede Aufgabe erhalten Sie von der oder die Betreuer*in das Feedback, ob die Ziele erreicht worden sind und Gegebenenfalls Tipps welche Ihnen helfen soll, die Punkte noch einmal zu fokusieren, die für Sie noch nicht genug verinnerlicht haben könnten.

# Ausgang Konseqzuenz Feedback
1

Ziele Erfüllt

Keine Konsequenzen Der oder die Betreuer*in werden Sie darüber in Kenntnis setzen, dass das Aufgabenziel erfüllt wurde. Gegebenenfalls werden Ihnen noch Tipps gegeben wie die Aufgabe anderes bzw. besser gelöst werden könnten.
2

Ziele nicht erfüllt

 Nachbearbeitung Im Falle, dass Sie nicht alle Ziele erfüllt haben sollten, wird Ihnen der Betreuer mitteilen welche Aspekte der Abgabe nicht zufriedenstellend sind. Sie haben anschließend drei Tage Zeit die Abgabe zu verbessern. Ergibt sich nach den drei Tagen eine Verbesserung jedoch noch immer kein zufriedenstellendes Erebnis, erhalten Sie noch eine weitere Nachbesserung. Falls dies anschließend wieder nicht zufriedenstellend ist, ist anzuzweifeln ob Sie für die Endabgabe die Inhalte außreichend verinnerlicht haben.
3

Keine Abgabe

 unter vorberhalt
von der praktischen Prüfung ausgeschlossen		 Falls Sie eine Aufgabe nicht hochladen, gibt es keine Nachbearbeitung. Hiebei ist jedoch wichtig zu erwähnen, dass dies nur dann passiert falls folgende Kriterien erfüllt sind:
  • Sie haben die Aufgabe unbegründet nicht abgegeben
  • Es ist kein technischer Fehler aufgetreten (z.B.: Probleme bei der Steuerung mit Git)
  • Sie haben nicht versucht mit uns Kontakt aufzunehmen

Die engültige Entscheidung eines Ausschlusses der praktischen Prüfung obligt dem Lehrbeauftragen bzw. Professor. Die Aufgabe der Betreuer liegt darin, Ihnen bei den Aufgaben zu unterstützen und nicht Sie zu bewerten.
Bedenken Sie, dass bei dieser Liste der persönliche Aspekt nicht mit eingeflossen ist. Somit ist die Kommunikation das Wichstigste bei Schwierigkeiten und Probleme. Die betreuenden Person steht es frei auch weitere Nachbearbeitungen anzubieten und den Studierenden in allen anderen Aspekten entgegen zu kommen, solange es zum Erfolg führt.

Abgabe

Die Abgabe erfolgt über den Felix-Kurs. Hier laden Sie in einem Textdokument hoch, welches Folgende Informationen enthält:

  • Link zu Ihrem Repository
  • Link zu Ihrem Github-Pages allgemein
  • Link zu Ihrer entsrpechenden Aufgabe

eine solche Abgabe könnte folgendermaßen aussehen: Falls Sie beim Hochladen nicht den Felix-Editor benutzen, achten Sie darauf eine Datei hochzuladen, die sowohl mit MacOS und Windows angezeigt werden kann. Verwenden Sie kein Word oder OpenOffice. Empfohlen wird eine PDF-Datei.
